# Content Module

> This module is appended to CLAUDE.md during setup when the user's role is Content Writer / Content Strategist.

---

## Content-Specific Workflows

### Workflow — Content Pipeline

Maintain `research/content-pipeline.md` with:
- Content piece title
- Type (blog, landing page, email, case study, whitepaper, social post)
- Target keyword (if SEO-driven)
- Status (Ideation, Brief Ready, In Progress, In Review, Published)
- Owner
- Due date
- Published URL (when live)

### Workflow — Content Brief from Keyword

When the user says "brief for [topic/keyword]":
1. Draft a structured brief: title options, target audience, search intent, H2 structure, key points to cover, CTA, word count, tone, competitor references
2. Save to `research/content-briefs/[slug].md`
3. Add to content pipeline

### Workflow — Brand Voice Reference

Maintain `research/brand-voice.md` with:
- Tone descriptors (e.g., professional but approachable)
- Words to use / words to avoid
- Example sentences in the brand voice
- Client-approved messaging
- Competitor voice comparison

### Workflow — Content Performance

When the user shares content performance data:
- Log in `research/content-performance.md`
- Flag: high performers worth repurposing, underperformers to update or retire
- Track which topics/formats drive the most engagement

### Content-Specific Files

| File | Purpose |
|---|---|
| `research/content-pipeline.md` | Master content tracker |
| `research/content-briefs/` | Individual content briefs |
| `research/brand-voice.md` | Voice and tone reference |
| `research/content-performance.md` | Published content metrics |
| `research/editorial-calendar.md` | Monthly/quarterly publishing schedule |

### Content Reporting Additions

When producing meeting MOMs or status updates, always include:
- Content published since last meeting
- Content in pipeline (what's next)
- Client approvals needed
- Performance highlights from published content
